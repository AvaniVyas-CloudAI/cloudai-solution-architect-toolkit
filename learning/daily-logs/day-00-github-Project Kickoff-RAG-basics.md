## What I learned about MVP

MVP means Minimum Viable Product.

It is the smallest useful first version of a project.

The purpose of an MVP is not to build everything perfectly. The purpose is to create a simple working version that teaches something and can be improved over time.

For this project, the MVP will be a simple version of the tool that takes a business problem and generates a basic AI/cloud solution brief.

## What I learned about RAG

RAG stands for Retrieval-Augmented Generation.

In simple terms, RAG means the AI searches trusted documents before generating an answer.

A normal chatbot may answer using general model knowledge.

A RAG-based system first searches approved sources such as product documentation, FAQs, policies, support tickets, or internal guides. Then the AI uses that retrieved information to generate a more grounded answer.

## Simple RAG flow

User asks a question  
→ System searches trusted documents  
→ Relevant information is retrieved  
→ AI generates an answer using that information  
→ Answer is reviewed or shown to the user

## Why RAG matters

RAG is useful when the answer must be based on specific company knowledge.

For example, a support assistant should answer using the company’s actual product documentation instead of giving a generic answer.

This reduces the risk of incorrect or made-up responses.
